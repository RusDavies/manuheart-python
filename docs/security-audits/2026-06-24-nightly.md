## Finding: OSV repository manifest scan omitted the git root

- Severity: Medium / release-supply-chain scan coverage hardening
- Evidence: `scripts/check_osv_scanner.py` ran `osv-scanner scan source --recursive .`
  for the repository-manifest leg. With OSV Scanner 2.3.x, direct local testing
  showed the current git root is not included unless `--include-git-root` is
  passed.
- Impact: Runtime and tooling dependency lockfile scans still passed, so no
  vulnerable dependency was identified. The issue was weaker manifest-scan
  coverage than the wrapper and CI/release gate claimed.
- Fix: Added `--include-git-root` to the repository scan arguments.
- Regression guard: Added a unit test asserting the repository scan includes
  `--include-git-root`.
